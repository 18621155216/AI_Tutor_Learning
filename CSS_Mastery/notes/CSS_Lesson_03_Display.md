# Lesson 03: 显示模式 (Display) - 块级 vs 行内

在 HTML 中，每个元素都有一个默认的“显示模式”。理解这些模式，你就能明白为什么有些元素会自动换行，而有些元素却能挤在同一行。

## 1. 块级元素 (Block-level)
这些元素就像是网页里的“霸道总裁”。
- **特点**：
  1. **独占一行**：哪怕你只写了一个字，它也要霸占整行的宽度。
  2. **可以设置宽高**：你可以精准控制它的 `width` 和 `height`。
  3. **盒子模型全生效**：`padding`, `border`, `margin` 都能正常起作用。
- **代表标签**：`<div>`, `<h1>`~`<h6>`, `<p>`, `<ul>`, `<li>`。

## 2. 行内元素 (Inline)
这些元素就像是网页里的“排队路人”。
- **特点**：
  1. **不换行**：它们会一个接一个地排在同一行，直到排不下为止。
  2. **不能设置宽高**：给它写 `width: 100px` 是没用的，它的大小由内容撑开。
  3. **盒子模型半生效**：水平方向的 `padding` 和 `margin` 有效，但垂直方向（上下）的效果很诡异，通常不会撑开周围元素的距离。
- **代表标签**：`<span>`, `<a>`, `<strong>`, `<em>`。

## 3. 行内块元素 (Inline-block)
这是前两者的“混血儿”，非常实用。
- **特点**：
  1. **不换行**：像行内元素一样并排站。
  2. **可以设置宽高**：像块级元素一样听话。
- **常见用途**：导航栏、按钮、小卡片并排显示。

## 4. 如何转换模式？
使用 `display` 属性，你可以强行改变元素的性格：
```css
span { display: block; } /* 让小路人变成霸道总裁 */
div { display: inline; } /* 让霸道总裁去排队 */
```
