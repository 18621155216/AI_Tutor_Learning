# Lesson 05: 长度单位 - 死的 px 与活的 rem/vw

在之前的课程中，我们一直用 `px`（像素）来设置大小。但在现代网页开发中，为了让网页能适应各种屏幕（手机、平板、大屏幕电脑），我们需要学会使用“活的”相对单位。

## 1. 绝对单位：px (像素)
- **定义**：屏幕上的一个物理点。
- **特点**：它是“死的”。你设置 `200px`，它在任何屏幕上永远都是 `200px`。
- **缺点**：不灵活。如果用户在浏览器里调大了字体，`px` 设置的大小不会跟着变。

## 2. 相对单位 A：rem (根元素比例) - 最推荐！
- **定义**：相对于网页根元素（通常是 `<html>`）的字体大小。
- **计算**：浏览器默认 `1rem = 16px`。
- **优点**：**极易维护**。如果你想整体调大网页的所有间距和文字，只需要改一下根元素的字号，所有用 `rem` 的地方都会跟着等比例缩放。
- **例子**：`32px` 就是 `2rem`。

## 3. 相对单位 B：em (父元素比例)
- **定义**：相对于当前元素父级的字体大小。
- **缺点**：容易产生嵌套计算混乱，新手建议优先使用 `rem`。

## 4. 视口单位：vw 和 vh (屏幕比例)
- **vw (Viewport Width)**：视口宽度的 1%。`100vw` 就是满屏宽。
- **vh (Viewport Height)**：视口高度的 1%。`100vh` 就是满屏高。
- **用途**：常用于制作全屏背景图，或者让盒子的宽度随窗口大小自动伸缩。

## 5. 视觉挑战题 (Visual Challenge)

**任务**：感受“活”的单位。

1. 打开 `exercises/05_Units/index.html`。
2. 在 `style.css` 中完成以下任务：
    - 给 `.px-box` 设置 `width: 300px`。
    - 给 `.rem-box` 设置 `width: 18.75rem` (即 300px)。
    - 给 `.vw-box` 设置 `width: 30vw` (屏幕宽度的 30%)。
3. **关键实验**：
    - **拉动浏览器窗口宽度**：看看哪个盒子的大小变了，哪个没变？
    - **修改根元素字号**：在 CSS 顶部加上 `html { font-size: 20px; }`，看看 `.rem-box` 是不是变大了？而 `.px-box` 变了吗？
