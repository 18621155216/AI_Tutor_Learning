# Lesson 13: Web Accessibility (A11y) 实战

## 核心原理 (Principles)

### 1. 什么是 A11y？
- **目标**：确保残障人士（视力、听力、运动、认知障碍等）能够感知、理解、导航和交互你的网站。
- **重要性**：道德、法律要求，提升用户体验。

### 2. `aria-*` 属性：增强语义
- **作用**：为辅助技术（如屏幕阅读器）提供额外信息，弥补原生 HTML 语义不足。
- **常用属性**：
    - `role`：定义元素角色（如 `role="button"`）。
    - `aria-label`：为无可见文本的元素提供可读标签。
    - `aria-describedby`：提供元素的额外描述。
    - `aria-live`：用于动态内容更新，控制屏幕阅读器宣布方式。

### 3. 屏幕阅读器优化
- **语义化 HTML**：A11y 基石，正确使用 `h1-h6`、`nav`、`main`、`alt` 等。
- **`tabindex`**：控制键盘焦点顺序。
    - `tabindex="0"`：可聚焦，按文档顺序。
    - `tabindex="-1"`：可聚焦，但不能通过 Tab 导航。
    - **避免** `tabindex=">0"`，会打乱自然顺序。

### 4. 键盘导航
- 所有可交互元素都应通过 Tab 键聚焦，Enter/Space 键激活。
- 焦点指示器（`outline`）必须可见。

---

## 最佳实践 (Best Practices)
- **语义化优先**：尽可能使用原生 HTML 标签，`aria-*` 作为补充。
- **`alt` 属性**：所有 `img` 标签都应有 `alt`。
- **`label` 关联**：所有表单控件都应与 `label` 关联。
- **焦点可见**：确保键盘焦点始终可见。
- **动态内容**：使用 `aria-live` 确保屏幕阅读器能感知到动态更新。

---

## 浏览器解析机制简述
- 浏览器会构建一个 **可访问性树 (Accessibility Tree)**，其中包含了 DOM 树的结构和语义信息，以及 `aria-*` 属性提供的额外信息。
- 屏幕阅读器等辅助技术会读取这个可访问性树，将网页内容以语音或盲文形式呈现给用户。
