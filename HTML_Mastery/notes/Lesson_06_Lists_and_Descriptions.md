# Lesson 06: Lists & Descriptions (列表布局与描述性数据)

## 核心原理 (Principles)

### 1. `<ul>`：无序列表 (Unordered List)
- **语义**：项与项之间没有顺序关系（如导航栏、功能列表）。
- **约束**：子元素**必须**是 `<li>`。
- **默认样式**：小圆点。

### 2. `<ol>`：有序列表 (Ordered List)
- **语义**：项与项之间有明确的顺序（如操作步骤、排行榜）。
- **属性**：
    - `type`：数字、字母或罗马数字（建议使用 CSS `list-style-type` 代替）。
    - `start`：起始编号。
- **默认样式**：数字编号。

### 3. `<dl>`：描述列表 (Description List)
- **语义**：用于术语及其描述（如 FAQ、字典）。
- **核心结构**：
    - `<dt>` (Term)：术语。
    - `<dd>` (Details)：描述。
- **架构师提示**：可以一对多，也可以多对多。

---

## 最佳实践 (Best Practices)
- **语义优先**：不要因为想要小圆点就用 `ul`。如果内容是有顺序的，应使用 `ol` 并通过 CSS 更改样式。
- **嵌套规则**：`li` 是 `ul`/`ol` 的唯一合法直接子元素。**严禁在 `ul` 里直接写 `div` 或 `p`**。
- **导航栏规范**：现代网页的导航栏通常使用 `nav > ul > li > a` 这种深度语义化的嵌套结构。

---

## 浏览器解析机制简述
- 浏览器在解析列表标签时，会自动应用特定的 **User Agent Stylesheet**。
- 有序列表 (`ol`) 会根据 DOM 树中 `li` 的顺序自动计算索引值。
- 描述列表 (`dl`) 的 `dt` 和 `dd` 会产生特定的外边距偏移，以体现层次关系。
