# Lesson 02: Text Structuring Tags (文本结构化标签)

## 核心原理 (Principles)

### 1. `<div>`：块级容器 (Block Container)
- **本质**：块级元素，独占一行，可设置宽高、内外边距。
- **用途**：页面布局，组合其他元素。
- **语义**：无语义，纯粹的容器。

### 2. `<span>`：行内容器 (Inline Container)
- **本质**：行内元素，不独占一行，只能设置左右内外边距。
- **用途**：对文本局部样式化或操作。
- **语义**：无语义，纯粹的容器。

### 3. `<h1>` - `<h6>`：标题标签 (Headings)
- **本质**：块级元素，定义不同层级的标题。
- **语义**：强语义，表示标题或章节标题。
- **重要性**：
    - **SEO**：帮助搜索引擎理解内容结构。
    - **A11y**：辅助屏幕阅读器导航。
- **规范**：按顺序使用，不跳级。

### 4. `<p>`：段落标签 (Paragraph)
- **本质**：块级元素，定义文本段落。
- **语义**：表示独立文本段落。
- **规范**：不能嵌套块级元素，可嵌套行内元素。

### 5. `<br>`：换行标签 (Line Break)
- **本质**：空标签，插入强制换行符。
- **用途**：诗歌、地址等需要精确换行的场景。
- **注意**：不应滥用，间距应通过 CSS 控制。

### 6. `<hr>`：水平线标签 (Horizontal Rule)
- **本质**：空标签，插入水平线。
- **语义**：表示内容主题变化或分隔。

---

## 最佳实践 (Best Practices)
- 优先使用语义化标签，避免滥用 `div` 和 `span`。
- 使用 `h1-h6` 标签来构建文档大纲，而不是仅仅为了样式。
- 使用 `<p>` 标签来包裹段落文本。
- 避免使用 `<br>` 标签来创建段落间距。
- 使用 `<hr>` 标签来表示内容主题的明确分隔。

---

## 浏览器解析机制简述
- 浏览器根据标签的语义和类型（块级/行内）来构建渲染树，并决定元素的默认样式和布局行为。
- 语义化标签有助于浏览器更好地理解文档结构，从而进行更准确的渲染和优化。
